Create a new test file `src/mcp/tools.test.ts` that tests the **exported** `createRegenerationToolHandlers` function. This is the primary testable surface in `mcp/tools.ts` — it accepts an injected `RegenerationJobManager` interface and returns handler functions, making it ideal for unit testing with a mock manager.

**Do NOT test `registerMcpTools`** (it requires a real McpServer instance — that's integration-only). Focus exclusively on `createRegenerationToolHandlers` and the helper functions it exercises internally.

**Mock setup:**

Create a minimal mock `RegenerationJobManager` object:
```typescript
const mockManager = {
  trigger: vi.fn(),
  getStatus: vi.fn(),
};
```

**Required test cases:**

1. **`handleRegenerateDocs` — valid input, successful trigger** — call with `{ target: 'my-project' }`. Mock `trigger` to return `{ ok: true, job: { id: 'job-1', state: 'queued', target: { ref: 'my-project', resolved: '.' }, createdAt: '2026-03-01T00:00:00Z' }, dedupe: { joined: false, key: 'my-project', reason: 'none' }, guidance: { message: 'Poll status', pollAfterMs: 2000 } }`. Assert the returned payload has `structuredContent.ok === true`, `structuredContent.jobId === 'job-1'`, `structuredContent.state === 'queued'`, and `structuredContent.next.tool === 'regenerate_docs_status'`.

2. **`handleRegenerateDocs` — invalid input** — call with `{}` (empty, no target is actually valid since target is optional) or call with `{ target: '', extraField: true }` (strict mode rejects extra fields). Assert the returned payload has `isError: true` and `structuredContent.ok === false` with a structured error.

3. **`handleRegenerateDocs` — trigger failure** — mock `trigger` to return `{ ok: false, error: { code: 'REGEN_IN_FLIGHT', reason: 'Already running', remediation: 'Wait for completion' }, guidance: { message: 'Wait and retry' } }`. Assert the returned payload has `isError: true` and the error code is propagated.

4. **`handleRegenerateDocsStatus` — valid input, successful status** — call with `{ jobId: 'job-1' }`. Mock `getStatus` to return a completed job. Assert `structuredContent.lifecycle.stage === 'completed'` and `structuredContent.lifecycle.progressPercent === 100`.

5. **`handleRegenerateDocsStatus` — invalid input (empty jobId)** — call with `{ jobId: '' }`. Assert validation error returned.

6. **`handleRegenerateDocsStatus` — job not found** — mock `getStatus` to return `{ ok: false, error: { ... }, guidance: { ... } }`. Assert error payload.

**Assertions must verify actual `structuredContent` shape** (the returned object), not just that mocks were called. Use `expect(result.structuredContent).toEqual(expect.objectContaining({ ... }))` or field-level assertions.

Follow project conventions: colocated test file at `src/mcp/tools.test.ts`, import from `vitest`, use `vi.fn()` for the mock manager.

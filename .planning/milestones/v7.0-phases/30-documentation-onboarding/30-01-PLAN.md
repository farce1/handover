<task type="auto">
  <name>Task 2: Add --yes flag and TTY guard to handover init</name>
  <files>src/cli/init.ts, src/cli/index.ts</files>
  <action>
**In `src/cli/index.ts`:** Add `--yes` option to the `init` command. Change the init command registration from:
```typescript
program.command('init').description('Create .handover.yml configuration file').action(runInit);
```
to:
```typescript
program
  .command('init')
  .description('Create .handover.yml configuration file')
  .option('--yes', 'Skip prompts and write a default config (for CI/non-interactive environments)')
  .action(runInit);
```

**In `src/cli/init.ts`:** Modify `runInit()` to accept an options object and add non-interactive handling. Follow the existing `@clack/prompts` pattern from `src/cli/generate.ts` for `isCI`/`isTTY` imports:

1. Add imports at the top:
   ```typescript
   import { isCI, isTTY } from '@clack/prompts';
   ```

2. Change the function signature from `export async function runInit(): Promise<void>` to:
   ```typescript
   export async function runInit(options: { yes?: boolean } = {}): Promise<void> {
   ```

3. At the very start of the function body (before the existing `p.intro(...)` line), add the TTY guard and `--yes` mode branches:

   ```typescript
   const isInteractive = isTTY(process.stdout) && !isCI();

   // Non-interactive without --yes: exit with instruction
   if (!isInteractive && !options.yes) {
     process.stderr.write(
       'handover init: non-interactive environment detected.\n' +
       'Run with --yes to create a default configuration file non-interactively.\n'
     );
     process.exit(1);
     return;
   }

   // Overwrite guard: --yes never silently overwrites
   if (existsSync('.handover.yml') && options.yes) {
     process.stdout.write('.handover.yml already exists — skipping.\n');
     return;
   }

   // Non-interactive --yes mode: write default config
   if (options.yes) {
     const defaultConfig = { provider: 'anthropic' as const };
     const validated = HandoverConfigSchema.parse(defaultConfig);
     const yamlContent = [
       '# handover configuration',
       '# Generated by `handover init --yes`',
       `# Docs: https://github.com/handover/handover`,
       '',
       stringifyYaml(defaultConfig, { lineWidth: 80 }),
     ].join('\n');

     writeFileSync('.handover.yml', yamlContent, 'utf-8');
     process.stdout.write('Created .handover.yml with default configuration.\n');
     process.stdout.write('Run handover generate to analyze your codebase.\n');
     return;
   }
   ```

4. The existing interactive flow (from `p.intro(...)` onward) remains unchanged — it only runs when `isInteractive` is true and `--yes` is not set.

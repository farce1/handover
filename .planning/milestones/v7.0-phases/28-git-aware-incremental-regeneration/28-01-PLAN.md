  <name>Git Fingerprint Module</name>
  <files>src/cache/git-fingerprint.ts, src/cache/git-fingerprint.test.ts, vitest.config.ts</files>
  <behavior>
    The module exports a discriminated union type and an async function:

    ```typescript
    export type GitFingerprintResult =
      | { kind: 'ok'; changedFiles: Set<string> }
      | { kind: 'fallback'; reason: string };

    export async function getGitChangedFiles(
      rootDir: string,
      sinceRef: string,
    ): Promise<GitFingerprintResult>
    ```

    Test cases (input -> expected output):

    **Happy path — committed + uncommitted changes:**
    - diffSummary returns files [a.ts, b.ts], status has modified [c.ts], not_added [d.ts]
    - -> { kind: 'ok', changedFiles: Set(['a.ts', 'b.ts', 'c.ts', 'd.ts']) }

    **Happy path — zero changes:**
    - diffSummary returns empty, status returns clean
    - -> { kind: 'ok', changedFiles: Set([]) }

    **Fallback — not a git repo:**
    - checkIsRepo returns false
    - -> { kind: 'fallback', reason: 'Not a git repo' }

    **Fallback — shallow clone:**
    - checkIsRepo true, raw('rev-parse --is-shallow-repository') returns 'true\n'
    - -> { kind: 'fallback', reason: 'Shallow clone detected' }

    **Fallback — detached HEAD:**
    - checkIsRepo true, not shallow, status.detached = true
    - -> { kind: 'fallback', reason: 'Detached HEAD' }

    **Error — invalid ref:**
    - revparse throws GitError for bad ref
    - -> throws Error with message containing the bad ref name

    **Error — revparse returns empty string:**
    - revparse returns '' (some git versions don't throw)
    - -> throws Error with message containing the bad ref name

    **Edge case — staged + created + deleted + renamed files all included:**
    - status has created [e.ts], deleted [f.ts], staged [g.ts], renamed [{to: 'h.ts'}]
    - -> all appear in changedFiles set

    **Edge case — shallow detection failure (old git):**
    - raw('rev-parse --is-shallow-repository') throws
    - -> treated as non-shallow, continues to status check

    **Edge case — deduplicated files:**
    - same file appears in both diffSummary and status.modified
    - -> appears once in Set (Set handles dedup automatically)
  </behavior>
  <implementation>
    1. Import `simpleGit` and `GitError` from 'simple-git'.
    2. Create git instance with `simpleGit(rootDir)`.
    3. Check sequence (order matters — each check can fail if prior conditions not met):
       a. `checkIsRepo()` — if false, return fallback 'Not a git repo'
       b. `git.raw(['rev-parse', '--is-shallow-repository'])` — if 'true', return fallback 'Shallow clone detected'. Wrap in try/catch for old git compatibility.
       c. `git.status()` — if `status.detached`, return fallback 'Detached HEAD'
       d. `git.revparse([sinceRef])` — validate ref exists. If GitError thrown OR empty string returned, throw Error (not fallback). Include ref name in error message.
       e. `git.diffSummary([sinceRef, 'HEAD'])` — get committed changes
       f. Union committed files + all status arrays: modified, created, deleted, renamed.map(r => r.to), not_added, staged
       g. Return { kind: 'ok', changedFiles: new Set(union) }

    IMPORTANT ordering note: step (c) calls git.status() which is also needed for step (f). Store the status result and reuse it — do NOT call git.status() twice.

    The ref validation (step d) happens AFTER repo/shallow/detached checks because:
    - Per CONTEXT.md locked decision: invalid ref is a user error (exit non-zero), but missing git context is a graceful fallback (exit 0)
    - If we validated ref first in a non-git repo, the error would be confusing

    Also update vitest.config.ts: replace `'src/cache/**'` with `'src/cache/round-cache.ts'` and add justification comment for git-fingerprint.ts being mock-testable.
  </implementation>
</feature>

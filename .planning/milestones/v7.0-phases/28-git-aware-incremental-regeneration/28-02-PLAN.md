    **src/cli/generate.ts** — In the `static-analysis` step's `execute` callback, AFTER the existing `analysisFingerprint` computation (line ~481) and BEFORE the `AnalysisCache` incremental detection block (line ~490), add the git-aware incremental logic:

    1. Add import at top of file:
       ```typescript
       import { getGitChangedFiles } from '../cache/git-fingerprint.js';
       ```

    2. After `analysisFingerprint = RoundCache.computeAnalysisFingerprint(fileEntries);` and the verbose log, add:
       ```typescript
       // ─── Git-aware incremental mode (--since) ──────────────────────────
       let gitChangedFiles: Set<string> | undefined;
       let isGitIncremental = false;

       if (options.since) {
         const gitResult = await getGitChangedFiles(rootDir, options.since);

         if (gitResult.kind === 'fallback') {
           // Specific warning per scenario, then fall back to content-hash mode
           logger.warn(`${gitResult.reason} — falling back to content-hash mode`);
           // isGitIncremental stays false; proceed with normal content-hash path below
         } else {
           // Zero changed files → early exit per locked decision
           if (gitResult.changedFiles.size === 0) {
             process.stdout.write(
               `No files changed since ${options.since} — nothing to regenerate\n`,
             );
             renderer.destroy();
             process.exit(0);
           }

           gitChangedFiles = gitResult.changedFiles;
           isGitIncremental = true;
         }
       }
       ```

    3. IMPORTANT: The git-aware path must OVERRIDE the content-hash incremental detection. Replace the existing AnalysisCache-based incremental block with a conditional:
       - If `isGitIncremental` is true: use `gitChangedFiles` as the `changedFiles` set for context packing and display. Set `isIncremental = true`, `changedFiles = gitChangedFiles`.
       - If `isGitIncremental` is false: use the existing `AnalysisCache`-based detection unchanged.

       The modified block should look like:
       ```typescript
       let changedFiles: Set<string>;
       let isIncremental: boolean;

       if (isGitIncremental && gitChangedFiles) {
         // Git-aware mode: changedFiles from git diff+status
         changedFiles = gitChangedFiles;
         isIncremental = true;
       } else {
         // Content-hash mode: detect changes from AnalysisCache (existing logic)
         const analysisCache = new AnalysisCache(
           join(rootDir, '.handover', 'cache', 'analysis.json'),
         );
         await analysisCache.load();

         const currentHashes = new Map<string, string>();
         for (const entry of fileEntries) {
           if (entry.contentHash) {
             currentHashes.set(entry.path, entry.contentHash);
           }
         }

         changedFiles = analysisCache.getChangedFiles(currentHashes);
         isIncremental = analysisCache.size > 0 && changedFiles.size < currentHashes.size;

         // Update analysis cache for next run's change detection
         for (const [path, hash] of currentHashes) {
           analysisCache.update(path, hash);
         }
         await analysisCache.save();
       }
       ```

       Note: When in git-aware mode, we still run AnalysisCache save for the NEXT content-hash run. Move the cache save outside the else block — always save current hashes regardless of mode. This way content-hash mode is not broken by a prior --since run.

    4. Update display state for incremental metadata. Use the same `displayState.isIncremental`, `changedFileCount`, `unchangedFileCount` fields. Additionally set `displayState.sinceRef`:
       ```typescript
       if (isIncremental) {
         displayState.isIncremental = true;
         displayState.changedFileCount = changedFiles.size;
         displayState.unchangedFileCount = currentHashes.size - changedFiles.size;
       }
       if (options.since && isGitIncremental) {
         displayState.sinceRef = options.since;
       }
       ```

    5. The `analysisFingerprint` computation stays UNCHANGED (computed from full fileEntries). Per research recommendation: the fingerprint still needs all files for round cache validity. The git-derived `changedFiles` set only affects which files get packed into context (the `packFiles` call).

    6. Error handling for invalid ref: `getGitChangedFiles` throws on invalid ref. This throw will propagate up to `handleCliError` in the CLI entry point, which prints the error message and exits non-zero. This satisfies the locked decision without additional catch logic in generate.ts.
